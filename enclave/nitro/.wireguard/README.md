The use of Wireguard with AWS Nitro Enclaves allows for the wg0 network interface to be used such that HTTPS traffic can occur between the enclave and the internet. Specifically, the `wireguard-go-vsock` binary built from the packages [source code](https://github.com/seedcx/wireguard-go-vsock) enables the wg0 network interface to be provided to the enclave over vsock through the host/parent instance. 

Although traffic is encrypted between the host and enclave using the Wireguard credentials contained in this folder, the packets are immediately decrypted by the host before it routes them to the outside internet. HTTPS is relied upon for confidentiality of request data being sent by the enclave, not Wireguard, which is used for convenience to forward traffic over vsock using the wg0 interface. Therefore, the Wireguard keys in this folder do not need to be confidential, and in fact are deliberately not concealed as to ensure the enclave build can be reproducible.
